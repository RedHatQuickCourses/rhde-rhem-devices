= Deploying Red Hat Edge Manager and Managing Edge Devices
:navtitle: Home

== Goal
This course covers the role of Red Hat Edge Manager by itself and relative to other infrastructure management tools from Red Hat.
This course also explains how to deploy Edge Manager on Red Hat Enterprise Linux (RHEL) servers from RPM packages; and enroll, onboard, and manage individual edge devices running Red Hat Device Edge and image mode for RHEL.

It is the first in a series of quick courses about Red Hat Edge Manager, which will consist of two courses:

. *Deploying Red Hat Edge Manager and Managing Edge Devices* (this course)

. https://TBD[Managing Edge Device Fleets and Applications with Red Hat Edge Manager] (follow-up course)

IMPORTANT: Red Hat associates must enroll using the https://training-lms.redhat.com/sso/saml/auth/rhlpint?RelayState=deeplinkoffering%TBD[Red Hat Learning Portal], and Red Hat Partners must enroll using https://training-lms.redhat.com/sso/saml/auth/rhopen?RelayState=deeplinkoffering%TBD[Red Hat Partner Connect].
Red Hat employees and partners must also enroll on HOL019 by following the links in the _Classroom environment_ heading bellow

== Objectives

On completing this course, you should be able to:

* Understand the purpose of Red Hat Edge Manager and how it relates to other infrastructure management tools from Red Hat.

* Deploy Red Hat Edge Manager on Red Hat Enterprise Linux (RHEL).

* Enroll managed devices on Red Hat Edge Manager.

* Onboard, update, and repurpose managed devices.

* Decommission managed devices.

== Audience

Operations-focused customer roles, who manage edge devices, but are not usually in-person at edge sites:

* System Administrators (IT - Information Technology, and OT - Operations Technology)
* Site Reliability Engineers (SREs)
* Platform Engineers
* DevOps Engineers

Red Hat and Partner roles that perform demonstrations, PoC and PoV of Red Hat Edge Manager as part of a sales motion, such as Solutions Architects.

Red Hat and Partner roles that support customers of Red Hat Edge Manager, such as Services Consultants and Customer Support Engineers.

== Prerequisites

This course assumes that you have the following experience:

* Familiarity with RHEL system administration, to the level of RHCSA (Red Hat Certified System Administrator) or equivalent knowledge.

* Familiarity with Linux container tools, such as Podman, and technologies compatible with Open Container Initiative (OCI) specifications, such as container registries.

* Familiarity with image mode for RHEL, especially building bootc container images and provisioning and updating image mode systems.
It's recommended that learners already prossess skills from the https://redhatquickcourses.github.io/rhde-bootc-build/[image mode with Red Hat Device Edge quick course series].
Notice that no prior knowledge of Kubernetes and OpenShift is assumed or required.
