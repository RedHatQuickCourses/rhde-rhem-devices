Deploying Red Hat Edge Manager and Managing Edge Devices

Goal

This course covers the role of Red Hat Edge Manager (RHEM) by itself and relative to other infrastructure management tools from Red Hat. This course also explains how to deploy RHEM on Red Hat Enterprise Linux (RHEL) servers from RPM packages; and enroll, onboard, and manage individual edge devices running Red Hat Device Edge and image mode for RHEL.

It is the first in a series of quick courses about Red Hat Edge Manager (RHEM), which will consist of two courses:

  1. Deploying Red Hat Edge Manager and Managing Edge Devices (this course)

  2. Managing Edge Device Fleets and Applications with Red Hat Edge Manager (follow-up course)

Red Hat associates must enroll using the Red Hat Learning Portal, and Red Hat Partners must enroll using Red Hat Partner Connect. Red Hat employees and partners must also enroll on HOL019 by following the links in the Classroom environment heading bellow

Objectives

On completing this course, you should be able to:

  • Understand the purpose of Red Hat Edge Manager and how it relates to other infrastructure management tools from Red Hat.

  • Deploy Red Hat Edge Manager on Red Hat Enterprise Linux (RHEL).

  • Enroll managed devices on Red Hat Edge Manager.

  • Onboard, update, and repurpose managed devices.

  • Decommission managed devices.

Audience

Operations-focused customer roles, who manage edge devices, but are not usually in-person at edge sites:

  • System Administrators (IT - Information Technology, and OT - Operations Technology)

  • Site Reliability Engineers (SREs)

  • Platform Engineers

  • DevOps Engineers

Red Hat and Partner roles that perform demonstrations, PoC and PoV of Red Hat Edge Manager as part of a sales motion, such as Solutions Architects.

Red Hat and Partner roles that support customers of Red Hat Edge Manager, such as Services Consultants and Customer Support Engineers.

Prerequisites

This course assumes that you have the following experience:

  • Familiarity with RHEL system administration, to the level of RHCSA (Red Hat Certified System Administrator) or equivalent knowledge.

  • Familiarity with Linux container tools, such as Podman, and technologies compatible with Open Container Initiative (OCI) specifications, such as container registries.

  • Familiarity with image mode for RHEL, especially building bootc container images and provisioning and updating image mode systems. It’s recommended that learners already prossess skills from the image mode with Red Hat Device Edge quick course series. Notice that no prior knowledge of Kubernetes and OpenShift is assumed or required.

For all learners

After you enter HOL019 on ROLE, follow the instructions there to create a new course classroom lab and access its virtual machines (VMs). If you already created a HOL019 classroom for another image mode with Red Hat Device Edge quick course, you can continue using the same classroom lab.

Be aware that ROLE classrooms have timers after which classroom labs are stopped and deleted, to conserve compute resources. If you plan to use the same classroom lab for multiple Red Hat Device Edge quick courses, do not wait too long to continue.

You do not need to use the same classroom for multiple courses: each course provides step-by-step instructions to start from either a new classroom lab or to continue from an existing classroom lab.

You will follow this quick course by alternating between two browser windows: one to navigate quick course materials, such as this page; and another to access the virtual classroom lab environment, to perform hands-on activities.

If you can provision your own RHEL servers or VMs, it should be possible to configure all that is required for the hands-on activities by using a free subscription from the Red Hat Developer Program, which gives access to RHEL installation media and RHEL package repositories.
