Lab: Install Red Hat Edge Manager on RHEL

Estimated reading time: 5 minutes.

Objective

Deploy Edge Manager on RHEL with minimal requirements.

Before you begin

You need a development machine running RHEL, to which you have unrestricted sudo access, and enough capacity to run a few VMs. Your development machine is already registered to Red Hat with a valid RHDE subscription by the first lab.

You also need a server machine running RHEL, to which you also have unrestricted sudo access.

Both your development machine and server machine require internet access to download RPM packages and container images.

If you are using the course classroom, your development machine is the workstation VM. Log in as the user student with password student. From that machine, you may start SSH sessions to the servera VM, which is your server machine, using the same user. Alternatively, you can open a shell or a console directly on the servera VM.

Instructions

You will install and configure Edge Manager on your server machine, configure a couple users with different roles, and then you will use your development machine to run the web UI and the CLI of Edge Manager and verify that your new Edge Manager instance is healthy.

The instructions here use the community artifacts from the Flight Control project, but are expected to work with the product artifacts from Edge Manager, when they become available, by just switching from the community repository to the Edge Manager product repository and using the Red Hat subscription manager to register your server machine.

  1. Verify that you have the outcomes of previous labs. If you do not, go back and complete them, because you need their outcomes to continue with this course.

    1. On your development machine, verify that there is a clone of the course samples repository on your home directory.

      $ cd
$ ls rhde-rhem-samples
LICENSE  README.md  antora.yml  enroll  modules  webapp

    2. Verify that your development machine is registered to Red Hat, so it can access package repositories of any RHEL version.

      $ sudo subscription-manager status
+-------------------------------------------+
   System Status Details
+-------------------------------------------+
Overall Status: Registered

    3. Other preparation tasks performed during the first lab of this course are not required yet, because most of this lab happens on the server machine, but they will be necessary for the remaining labs.

  2. Install the Edge Manager services on your server machine.

    1. Make sure your current shell is on the right machine.

      $ hostname
servera

    2. Install the container tools on your server machine because the Edge Manager packages just configure Systemd to run the Edge Manager services as quadlets, that is, as Podman containers.

      $ sudo dnf install podman skopeo
...
Complete!

    3. Enable the Flight Control repository from EPEL.

      $ sudo dnf config-manager --add-repo https://rpm.flightctl.io/flightctl-epel.repo
Adding repo from: https://rpm.flightctl.io/flightctl-epel.repo
      There is no need to register servera because the Flight Control services package contains no dependencies other than a container engine (Podman) and the classroom environment contains a mirror of the RHEL 10 packages, which the sererva machine is preconfigured to use.

    4. Install the Edge Manager services.

      $ sudo dnf install -y flightctl-services-1.0.0
...
Complete!
      If you want to try the current latest release from the upstream project, remove the version component -1.0.0 from the package flightctl-services in the command above.

    5. Start the Edge Manager services.

      $ sudo systemctl enable --now flightctl.target
Created symlink '/etc/systemd/system/multi-user.target.wants/flightctl.target' → '/usr/lib/systemd/system/flightctl.target'.

      $ sudo podman logs flightctl-api

  3. For simplicity, configure local users for the Edge Manager services, on the PAM issuer container.

    1. Create groups which correspond to the admin, operator, and viewer roles.

      $ sudo podman exec -i flightctl-pam-issuer groupadd flightctl-admin
$ sudo podman exec -i flightctl-pam-issuer groupadd flightctl-operator
$ sudo podman exec -i flightctl-pam-issuer groupadd flightctl-viewer

    2. Create a user named student and grant it the operator role.

      $ sudo podman exec flightctl-pam-issuer adduser student
$ sudo podman exec -i flightctl-pam-issuer sh -c 'echo "student:student" | chpasswd'
$ sudo podman exec -i flightctl-pam-issuer usermod -aG flightctl-operator student

    3. Create a user named admin and grant it the admin role.

      $ sudo podman exec flightctl-pam-issuer adduser admin
$ sudo podman exec -i flightctl-pam-issuer sh -c 'echo "admin:redhat" | chpasswd'
$ sudo podman exec -i flightctl-pam-issuer usermod -aG flightctl-admin admin

    4. Verify you have the required users and groups on the PAM issuer container.

      $ sudo podman exec -i flightctl-pam-issuer groups student
student : student flightctl-operator
$ sudo podman exec -i flightctl-pam-issuer groups admin
admin : admin flightctl-admin

    5. If you would rather be more thorough verifying users and groups on the PAM issue container, use the following commands.

      $ sudo podman exec -i flightctl-pam-issuer getent passwd student
student:x:1000:1003::/home/student:/bin/bash
$ sudo podman exec -i flightctl-pam-issuer getent passwd admin
admin:x:1001:1004::/home/admin:/bin/bash
$ sudo podman exec -i flightctl-pam-issuer getent group flightctl-admin
flightctl-admin:x:1000:admin
$ sudo podman exec -i flightctl-pam-issuer getent group flightctl-operator
flightctl-operator:x:1001:student

  4. Install the Edge Manager CLI on your development machine

    1. Make sure your current shell is on the right machine.

      $ hostname
workstation

    2. Enable the flight repository from EPEL and install its CLI package.

      $ sudo dnf config-manager --add-repo https://rpm.flightctl.io/flightctl-epel.repo
Adding repo from: https://rpm.flightctl.io/flightctl-epel.repo
$ sudo dnf install flightctl-cli
...
Complete!

    3. Verify that the flightctl command is in your command PATH.

      $ flightctl version
Client Version: v1.0.0
Server Version: Could not read server version
You must log in to view the server version. Please use the 'login' command to authenticate before proceeding

    4. Copy the self-signed certificate of the Edge Manager server and add it to the system’s trusted CA set.

      $ scp servera:/etc/flightctl/pki/ca-bundle.crt ~
ca-bundle.crt             100% 1222   128.6KB/s   00:00
$ sudo cp ca-bundle.crt /etc/pki/ca-trust/source/anchors/
$ sudo update-ca-trust extract

    5. Log in on the Edge Manager server.

      $ flightctl login -u admin -p redhat https://servera.lab.example.com:3443
Auto-selected organization: 00000000-0000-0000-0000-000000000000 Default
Login successful.

    6. Verify that the CLI can now get the server’s version, and that there are no managed devices yet.

      $ flightctl version
Client Version: v1.0.0
Server Version: v1.0.0
$ flightctl get device
NAME    ALIAS   OWNER   SYSTEM  UPDATED APPLICATIONS

  5. Access the Edge Manager web UI.

    1. Open a web browser and visit https://servera.lab.example.com. You should get a login screen without any web browser warnings about certificate issues.

    2. Log in as the student user, with the password student.

      If you are iddle for too long on the login page, you may get a "CSRF error". If it happens, just try again.

    3. You should get the Edge Manager Overview page, with three gray circles because there are no managed devices on which to report a status summary.

    4. On the left panel, click Devices to verify that there are indeed no managed devices.
